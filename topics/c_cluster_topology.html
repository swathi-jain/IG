Cluster topology example

IBM® Rational® Focal Point™ can be set up on a clustered environment to enable failover and load balancing.
Figure 1. Cluster topology example

In this topology example, the environment includes several web clients, a load balancer, several nodes of Rational Focal Point, a database server, and a license server.

In a cluster setup, a user can enter URLs to log on to the nodes. With the load balancer, the nodes are hidden behind a common URL prefix, such as http://focalpoint.node.com/fp/. Instead of accessing the nodes directly, users access the load balancer, which routes the request to one of the nodes. After a session is set up on a node, all of the subsequent requests in that session are routed to the same node.

For information about the requirements for setting up a clustered environment, see Setting up clusters for Rational Focal Point
