Upgrading from Rational Focal Point version 5.4 releases

If you are upgrading from a release of Rational® Focal Point™ that is earlier than version 6.4.1, you must uninstall the version. Before you upgrade to the latest version of IBM® Rational Focal Point, you must create a backup of the database and some files.

Procedure

  1. Open your instance of Rational Focal Point and verify the view definitions on the Views page (Advanced > Verify Integrity > Views). Fix any broken view definitions that are limitations for incoming link attributes, including views in inactive or archived workspace.
  2. Create a backup of the report images. The report images are stored in these locations:
    • On Windows systems: Rational Focal Point Installation directory\images
    • On Linux systems: /usr/share/focalpoint/images
    • On Solaris systems: /opt/FOCAfp/share/images
  3. Note any custom business rules. The custom business rules files are stored in these locations:
    • On Windows systems: Rational Focal Point Installation directory\forumlas.
    • On Linux systems: /usr/share/focalpoint/formulas
    • On Solaris systems: /opt/FOCAfp/share/formulas
    Important: After you install the latest release of Rational Focal Point, for compatibility reasons, you must request your custom business rules from the IBM Rational Focal Point support team. Only the common business rules are included in the installation. The common business rules are ChoiceAttributeText, LinkByText, LinkInfo, LinkListAttributeSum, ListAttributeSum, MailImport, MatrixWriter, SetChoice, and TrafficLight. Incompatible business rules are ignored.
  4. Back up the database.
  5. Stop the application server and uninstall Rational Focal Point. To uninstall a release of Rational Focal Point that is earlier than version 6.4.1:
    1. On Windows systems, click Start > Control Panel > Add or Remove Program and uninstall the product.
    2. On Linux systems, enter the following command: rpm -ev focalpoint
    3. On Solaris systems, enter the following command: pkgrm FOCAfp
    Note: On UNIX systems, the Apache Tomcat application server is uninstalled when you uninstall IBM Rational Focal Point. Before you install the latest release, you must install the application server.
  6. Before you can install the latest release of Rational Focal Point, you must upgrade to Rational Focal Point version 6.4.0. Obtain version 6.4.0 from IBM Rational Support.
  7. Install version 6.4.0. For more information about installing the package, see the installation guide.
  8. Start the application server. Rational Focal Point is now running.
  9. In a web browser, type the URL of the Rational Focal Point login page. For example: http://localhost:8080/fp/servlet/Login.
  10. Log on to Rational Focal Point. A confirmation message about the database upgrade is displayed.
  11. To confirm the database update, click Yes. The database update can take some time depending on the size of the database. When the database is updated, the updated workspace is displayed.
  12. Restart the services, log on to Rational Focal Point, and migrate the baseline. For information about migrating baselines, see the help system.
  13. Stop the application service and uninstall version 6.4.0:
    • On Windows systems, click Start > Control Panel > Add or Remove Program and uninstall the product.
    • On Linux systems, enter the following command: rpm -ev focalpoint
    • On Solaris systems, enter the following command: pkgrm FOCAfp
    Note: On UNIX systems, the Apache Tomcat application server is uninstalled when you uninstall Rational Focal Point. Before you install the latest version, you must install the application server.

What to do next

Install the latest release of Rational Focal Point and then complete the tasks in Migrating data from earlier versions of Rational Focal Point.
