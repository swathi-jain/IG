Upgrading from a release of Rational Focal Point version 6.5

If you installed a release of IBM® Rational® Focal Point™ version 6.5, you can upgrade directly to the latest release.

About this task

To upgrade to the latest release, you must back up the database and some files. Then, you can upgrade the product.

Procedure

  1. Open your instance of Rational Focal Point and verify the view definitions on the Views page (Advanced > Verify Integrity > Views). Fix any broken view definitions that might be limitations for incoming link attributes, including views in inactive or archived workspaces.
  2. Note any custom business rules that you have used. Log on to Rational Focal Point and export the custom business rules.
    Important: After you upgrade to the latest release of Rational Focal Point, for compatibility reasons, you must request the custom business rules from the IBM Rational Focal Point support team. Only the common business rules are included in the installation. The common business rules are ChoiceAttributeText, LinkByText, LinkInfo, LinkListAttributeSum, ListAttributeSum, MailImport, MatrixWriter, SetChoice, and TrafficLight. Incompatible business rules are ignored.
  3. Back up the database.
  4. Stop the application server.
    Tip: Before you upgrade Rational Focal Point in the production environment, test the release with your data in a test environment.
  5. To upgrade directly to the latest release of Rational Focal Point, see Upgrading Rational Focal Point.

What to do next

After you upgrade to the latest release of Rational Focal Point, migrate the required data. For more information, see Migrating data from earlier versions of Rational Focal Point.
