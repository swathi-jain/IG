Upgrading from releases of Rational Focal Point version 6.0

If you are upgrading from a release of Rational® Focal Point™ that is earlier than version 6.4.1, you must uninstall the version. Before you upgrade to the latest version of IBM® Rational Focal Point, you must create a backup of the database and some files.

Procedure

  1. Open your instance of Rational Focal Point and verify the view definitions on the Views page (Advanced > Verify Integrity > Views). Fix any broken view definitions that are limitations for incoming link attributes, including views in inactive or archived workspace.
  2. Log on to Rational Focal Point and export the custom business rules.
    Important: After you install the latest release of Rational Focal Point, for compatibility reasons, you must request your custom business rules from the IBM Rational Focal Point support team. Only the common business rules are included in the installation. The common business rules are ChoiceAttributeText, LinkByText, LinkInfo, LinkListAttributeSum, ListAttributeSum, MailImport, MatrixWriter, SetChoice, and TrafficLight. Incompatible business rules are ignored.
  3. Back up the database.
  4. Stop the application server and uninstall Rational Focal Point. To uninstall a release of Rational Focal Point that is earlier than version 6.4.1:
    1. On Windows systems, click Start > Control Panel > Add or Remove Program and uninstall the product.
    2. On Linux systems, enter the following command: rpm -ev focalpoint
    3. On Solaris systems, enter the following command: pkgrm FOCAfp
    Note: On UNIX systems, the Apache Tomcat application server is uninstalled when you uninstall IBM Rational Focal Point. Before you install the latest release, you must install the application server.

What to do next

Install the latest release of Rational Focal Point and then complete the tasks in Migrating data from earlier versions of Rational Focal Point.
